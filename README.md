# 🧶 Научный клубок — карта знаний R&D

GraphRAG-система для горно-металлургической отрасли: превращает разрозненные PDF/DOCX/PPTX/XLSX
(обзоры, статьи, доклады, журналы) в граф знаний Neo4j с семантическим поиском на естественном языке.

## Подключаемые LLM-модели в llama.cpp

https://huggingface.co/unsloth/Qwen3.6-35B-A3B-GGUF

https://huggingface.co/unsloth/Qwen3.6-27B-MTP-GGUF

https://huggingface.co/gpustack/bge-m3-GGUF

## Возможности

- **Поиск на естественном языке** — «Какие технические решения организации циркуляции католита
  при электроэкстракции никеля описаны в мировой практике?» → структурированный ответ со ссылками
  на источники [N] (документ, страница, год). Стриминг с кнопкой «Стоп», вкладки диалогов,
  экспорт отчёта в Markdown.
- **Аналитика в каждом ответе** (автоматически): обзор источников с группировкой, консенсус
  и разногласия, оценка достоверности, пробелы знаний, рекомендации (эксперты, смежные темы);
  для сравнительных вопросов — таблица сравнения.
- **Граф знаний** — 9 типов сущностей (Material, Process, Equipment, Experiment, Facility,
  Topic, Expert, Parameter, Document), 8 типов связей (вкл. `contradicts` и `validated_by`)
  с числом подтверждающих источников и датой актуализации на каждом факте.
- **3D-визуализация** — цепочки «материал → процесс → оборудование → результат» (результаты —
  полусферы), красно-белые рёбра противоречий, ⚠-подсветка пробелов, эксперты и организации
  по теме запроса.
- **Vision** — страницы-сканы и (опционально, галочкой) все изображения документов распознаются
  VLM и попадают в индекс; к запросу можно приложить картинку (кнопка / drag-and-drop / Ctrl+V).
- **Числовые параметры** — 23 тыс. извлечённых ограничений: значение + единица + оператор
  («сульфаты ≤ 300 мг/л»).
- **География и время** — разметка RU / зарубежная практика, фильтр по годам.
- **Пробелы в знаниях** — слабо освещённые процессы, темы только в отечественной или только
  в зарубежной литературе; анти-галлюцинация: на вопрос вне корпуса система отвечает «данных нет».
- **Локальный контур** — полная работа без интернета: llama.cpp + GGUF (LLM, эмбеддер, vision
  через mmproj), управление моделями из UI, переключение облако↔локалка.

## Архитектура

```
PDF/DOCX/PPTX/XLSX/XLS → ингест (PyMuPDF, python-docx/pptx, openpyxl, xlrd) → чанки (~4000 симв.)
    + сканы и изображения → VLM (Gemini онлайн / Qwen mmproj локально) → текстовые чанки
    → LLM-экстракция сущностей/связей/параметров (DeepSeek V4 или локальная модель)
    → Neo4j: граф знаний + векторный индекс (bge-m3, 1024d) + fulltext (Lucene)
    → Запрос: LLM-разбор → векторный + fulltext поиск + обход графа → синтез ответа с цитатами
    → Веб-UI: FastAPI + ванильный JS (поиск со стримингом / 3D-граф / данные / модели / аудит)
```

## Масштаб проиндексированного корпуса

176 документов → 4 698 чанков → 43 тыс. сущностей, 34.9 тыс. связей (из них 192 противоречия),
23.2 тыс. числовых параметров, 3 034 эксперта, 7 076 организаций (1 749 связей «работает в»),
4 353 упомянутых эксперимента.

## Запуск

```bash
# 1. Зависимости
uv sync

# 2. Ключ API (routerai.ru или любой OpenAI-совместимый)
cp .env.example .env   # вписать ROUTERAI_API_KEY

# 3. Neo4j
docker compose up -d

# 4. Пайплайн данных (данные кейса положить в data/)
uv run python scripts/run_ingest.py      # парсинг + чанкинг
uv run python scripts/run_extract.py     # LLM-экстракция (долго, есть чекпоинт)
uv run python scripts/run_embed.py       # эмбеддинги
uv run python scripts/run_load.py        # загрузка в Neo4j

# 5. UI (http://localhost:8000)
uv run uvicorn sciknot.api:app --host 127.0.0.1 --port 8000
```

Дальше документы добавляются через вкладку «Данные» (инкрементально, дедуп по содержимому).
Локальные модели: скопировать `models.example.json` в `models.json`, указать пути
к llama-server и GGUF — управление из вкладки «Модели».

## Структура

- `src/sciknot/ingest/` — парсеры (PDF/DOCX/PPTX/XLSX/XLS) и чанкинг
- `src/sciknot/extract/` — LLM-экстракция (промпт-онтология в `extractor.py`)
- `src/sciknot/graph/` — схема Neo4j, загрузка, индексы
- `src/sciknot/search/` — гибридный поиск, синтез ответов, пробелы
- `src/sciknot/dataops.py` — загрузка через UI: дедуп, vision-распознавание, очистка
- `src/sciknot/localllm.py` — менеджер локальных моделей llama.cpp (слоты, конфиги запуска)
- `src/sciknot/api.py` + `src/sciknot/web/` — FastAPI-бэкенд и веб-интерфейс
- `scripts/` — пайплайн, доиндексация, генерация презентации

## Соответствие требованиям ТЗ

| Требование ТЗ | Реализация | Где посмотреть |
|---|---|---|
| Многопараметрические запросы (материал+процесс+условия+гео+время) | LLM-разбор запроса → сущности, фильтры географии и лет, RU/EN синонимы; изображение-вложение к вопросу (vision) | Поиск → «Как система поняла запрос» |
| Верификация: источник, достоверность, дата актуализации | Ссылки [N] (документ/страница/год); confidence связи = N источников; created_at/updated_at на фактах | Ответ; граф (hover по связи) |
| Числовые ограничения | 23 тыс. параметров: значение + единица + оператор («сульфаты ≤ 300 мг/л») | Экстракция → Parameter-узлы |
| NLP-пайплайн: сущности, связи, синонимы RU/EN | LLM-экстракция с онтологией в промпте | `src/sciknot/extract/extractor.py` |
| Граф: онтология + Cypher | Neo4j: 9 типов узлов (вкл. Experiment, Facility), 8 типов связей (вкл. contradicts, validated_by), вектор + fulltext индексы | `src/sciknot/graph/loader.py` |
| Визуализация: цепочки М→П→О→результат, противоречия, пробелы, эксперты и лаборатории | 3D-граф: типизированные связи, полусферы результатов, зебра-рёбра противоречий, ⚠-узлы слабого покрытия, эксперты и организации по теме | Вкладка «Граф знаний» |
| Аналитика: группировка, консенсус/разногласия, достоверность, пробелы, рекомендации | 6 обязательных разделов генерируются автоматически в каждом ответе | Любой запрос в «Поиске» |
| Сравнительные запросы | Автоматическая таблица сравнения для запросов «А vs Б», «RU vs мир» | Поиск |
| Выявление пробелов знаний | В каждом ответе + отдельный блок статистики по графу | Вкладка «Данные» |
| Эксперты и лаборатории | Раздельные типы Expert (люди) и Facility (организации), связь AFFILIATED_WITH из аффилиаций | Граф, ответы |
| Импорт данных | Загрузка PDF/DOCX/PPTX/XLSX/XLS через UI, инкрементальная индексация с прогрессом, дедуп по содержимому, vision-распознавание сканов и изображений, полная очистка | Вкладка «Данные» |
| Масштабируемость | Инкрементальный пайплайн (чекпоинты), новые домены — расширение промпт-онтологии | ниже |
| Мультиязычность | Корпус и запросы RU/EN, bge-m3 мультиязычный | — |
| Экспорт результатов | Отчёт в Markdown одной кнопкой | Поиск → «⬇ Отчёт .md» |
| Локальный контур | llama.cpp + GGUF: слоты LLM/эмбеддер, vision (mmproj), переключение облако↔локалка с автоостановкой локальной модели | Вкладка «Модели», `docs/local_llm.md` |

дашборд руководителя, экспорт PDF/JSON-LD, нормализация вариантов имён организаций.

## Масштабирование

Пайплайн инкрементальный (чекпоинты по `chunk_id`): новые категории документов добавляются
командой `run_ingest.py --categories "Журналы"` без пересборки существующего графа.
Новые домены = новые типы сущностей в промпте экстракции + метки в загрузчике.
